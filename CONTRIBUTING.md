# Contributing to IR2MQTT

First off, thank you for considering contributing to IR2MQTT! It's people like you that make open source such a great community.

## Project Status & Expectations

I am currently the **sole maintainer** of this project. I develop and maintain IR2MQTT in my free time as a hobby project.

Because of this, please adjust your expectations regarding response times:
- Issues and pull requests are very welcome!
- I will review them as soon as I can, but it might take a few days or even weeks depending on my schedule.
- Please be patient and understanding if I cannot implement a requested feature or fix a bug immediately.

## How Can You Contribute?

### Reporting Bugs & Requesting Features
If you find a bug or have an idea for a new feature, please open an issue! 
- Use the provided issue templates.
- Provide as much detail as possible (logs, steps to reproduce, screenshots).
- Check if a similar issue already exists before creating a new one.

### Submitting Pull Requests
If you want to contribute code:
1. Fork the repository.
2. Create a new branch for your feature or bugfix (`git checkout -b feature/my-awesome-feature`).
3. Make your changes and ensure they follow the existing coding style.
4. Run the tests locally (using `make test` or `make check`) to ensure nothing is broken.
5. Commit your changes with clear, descriptive commit messages.
6. Push to your fork and submit a Pull Request against the `main` branch.

I will review your PR when I have time. Please understand that I might request changes or decide not to merge a PR if it doesn't align with the project's goals or adds too much maintenance burden.

## Local Development
For instructions on how to set up the development environment, please refer to the [Local Setup & Architecture](docs/dev-setup.md) page in the documentation.

Thank you for your support and understanding!
